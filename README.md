# Manual Testing Project - OpenCart

## Project Overview

This repository contains a comprehensive manual testing framework for **OpenCart**, an open-source e-commerce platform. The project provides structured test documentation, scenarios, and reporting templates to ensure thorough quality assurance of the OpenCart application.

**Application Under Test:** [OpenCart Demo](https://demo.opencart.com/)

---

## Key Features

### 1. **Test Cases**
The repository includes a well-defined set of test cases covering various aspects of OpenCart:

- **User Management**
  - User Registration
  - Login & Logout
  - Forgot Password
  - My Account Management

- **Product Management**
  - Product Search
  - Product Display Page
  - Product Comparison
  - Category Pages

- **Shopping Experience**
  - Add to Cart
  - Wish List
  - Shopping Cart Management
  - Currencies

- **Order Processing**
  - Checkout Process
  - Order History
  - Downloads

- **Administrative Functions**
  - Menu Options
  - Footer Options
  - Contact Us Page

Test cases are **categorized by module and user role** for easy navigation and execution, making it simple for testers to focus on specific areas of functionality.

### 2. **Test Scenarios**
Detailed test scenarios provide **step-by-step instructions** for replicating specific user journeys and interactions within the OpenCart platform. These scenarios:

- Facilitate consistent and efficient testing across different testers
- Ensure thorough coverage of key functionalities
- Follow real-world user workflows
- Include both positive and negative test scenarios
- Cover end-to-end business processes

### 3. **Expected Results**
Every test case clearly defines **expected results** for each interaction, which helps testers:

- Identify bugs and defects
- Detect regressions in functionality
- Spot inconsistencies in OpenCart's behavior
- Validate that features work as intended
- Compare actual vs. expected outcomes

### 4. **Reporting**
The project encourages testers to document their findings using a **standardized template**. This includes:

- Logging bugs with detailed descriptions
- Attaching screenshots and visual evidence
- Providing steps to reproduce issues
- Categorizing defects by severity and priority
- Tracking defect status and resolution

## Getting Started

### Prerequisites
- Basic understanding of manual testing concepts
- Familiarity with e-commerce applications
- Access to [OpenCart Demo](https://demo.opencart.com/)
- Spreadsheet software (Excel, Google Sheets) for test case documentation

### How to Use This Repository

1. **Review Test Plan**
   - Start with the Test Plan document to understand the project scope
   - Familiarize yourself with test strategy and approach

2. **Explore Test Cases**
   - Navigate to test cases organized by module
   - Review test case structure and format
   - Understand preconditions and test data requirements

3. **Execute Test Scenarios**
   - Follow step-by-step instructions in test scenarios
   - Document actual results during execution
   - Compare with expected results

4. **Report Defects**
   - Use the standardized defect reporting template
   - Include all necessary details (steps, screenshots, environment)
   - Categorize and prioritize issues appropriately
